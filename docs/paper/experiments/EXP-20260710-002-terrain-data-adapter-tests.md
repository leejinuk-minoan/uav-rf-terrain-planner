# EXP-20260710-002 - terrain data adapter tests

## Date

2026-07-10

## Related Task / Issue / PR

Task 016B - Terrain Data Adapter Interface Scaffold

## Experiment Purpose

Verify that the terrain data adapter scaffold can represent paired DEM/DSM metadata, reject alignment problems, expose synthetic DEM/DSM samples, calculate surface deltas, and guard against private local path strings.

## Input Data

Synthetic in-memory terrain only.

## Scenario / Configuration

The tests use small synthetic DEM/DSM grids generated by existing project fixtures and metadata records created in test code.

## Method

Unit tests instantiate metadata dataclasses, run the metadata validator, exercise the synthetic adapter at valid and invalid indexes, and inspect the new adapter module for external geospatial import patterns.

## Expected Result

Metadata validation should pass for aligned DEM/DSM metadata and fail for CRS, resolution, shape, raster type, and private path issues. The synthetic adapter should return DEM/DSM MSL values, compute DSM minus DEM surface delta, and raise explicit out-of-bounds exceptions.

## Actual Result

Local verification passed in Task 016B.
