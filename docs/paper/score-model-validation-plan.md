# 점수식 보완 및 검증 계획

작성일: 2026-07-08  
개정일: 2026-07-08  
책임 주체: GPT Master Agent  
적용 범위: 연구·교육·시뮬레이션용 UAV RF Terrain Planner  
중요 제한: 실제 드론 운용, 실제 비행, 실시간 제어, 현장 통신 보장 검증, 실제 링크품질 보장 검증은 수행하지 않는다.

---

## 1. 목적

본 문서는 발진 가능구역 종합점수와 차폐안정성 점수의 타당성을 보완·검증하기 위한 단계별 계획이다.

현재 점수식은 다음과 같다.

```text
발진 가능구역 종합점수 = 차폐안정성 점수 × 0.80 + 거리점수 × 0.20
차폐안정성 점수 = LOS 점수 × 0.50 + Fresnel 여유 점수 × 0.35 + DSM 장애물 점수 × 0.15
```

이 가중치는 실제 링크상태 데이터로 최적화된 값이 아니라 MVP 초기 설계 가정이다. 따라서 본 계획은 **실제 드론운용 없이 산출 가능한 데이터만 사용**하여 점수식을 보완한다.

본 계획의 목표는 다음이다.

1. 점수식을 모듈별로 분해한다.
2. 각 항목이 같은 방향성과 같은 범위의 점수로 정규화되는지 확인한다.
3. synthetic DEM/DSM에서 논리적 일관성을 검증한다.
4. 공개 또는 사용자 승인 샘플 DEM/DSM으로 오프라인 재현성을 확인한다.
5. 가중치 민감도 분석으로 결과의 안정성을 평가한다.
6. ablation 분석으로 각 점수항의 필요성을 검토한다.
7. 전파모델과의 방향성만 대조하고, 실제 수신성능을 입증했다고 표현하지 않는다.
8. 논문에서는 성능 입증이 아니라 방법론 검증과 한계로 표현한다.

## 2. 오프라인 데이터 사용 원칙

본 프로젝트의 점수식 보완은 다음 데이터만 사용한다.

| 데이터 구분 | 사용 여부 | 설명 | 논문 표현 |
|---|---:|---|---|
| Synthetic DEM | 사용 | 임의 생성 고도면 | 알고리즘 경계조건 검증 |
| Synthetic DSM | 사용 | 임의 생성 건물·수목·장애물 표면고도 | DSM 반영 로직 검증 |
| 공개 DEM | 사용 가능 | 공개 지형고도 자료 | 오프라인 지형자료 기반 재현성 |
| 공개 DSM | 사용 가능 | 공개 표면고도 자료 | 표면장애물 반영 검토 |
| 사용자 승인 샘플 DEM/DSM | 사용 가능 | 사용자가 제공하거나 사용을 승인한 샘플 | 사례지역 오프라인 검증 |
| 좌표·거리·고도 계산값 | 사용 | MGRS/WGS84/투영좌표, AGL/MSL, 3D 거리 | 입력·전처리 산출값 |

| Fresnel 침범률 | 사용 | 주파수·거리·장애물 단면 기반 계산값 | Fresnel clearance proxy |
| DSM 장애물 밀도/침범도 | 사용 | 표면고도 기반 계산값 | 표면장애물 proxy |
| RSSI | 사용 금지 | 실제 송수신 측정 필요 | 향후 연구 |
| SINR | 사용 금지 | 실제 간섭·잡음 측정 필요 | 향후 연구 |
| Packet loss | 사용 금지 | 실제 통신 실험 필요 | 향후 연구 |
| 실제 비행 로그 | 사용 금지 | 실제 드론운용 필요 | 본 연구 범위 제외 |
| 실제 통신 성공률 | 사용 금지 | 실제 운용·측정 필요 | 본 연구 범위 제외 |

## 3. 검증 원칙

| 원칙 | 설명 |
|---|---|
| 실제 운용 배제 | 실제 드론 비행, 실제 조종기 연결, 실시간 링크 측정은 수행하지 않는다. |
| 오프라인 산출값만 사용 | DEM/DSM, 좌표, 거리, 고도, LOS, Fresnel, DSM 장애물 proxy만 사용한다. |
| synthetic 우선 | 임의 생성 지형으로 알고리즘의 방향성과 경계조건을 먼저 확인한다. |
| 공개/샘플 데이터 사용 | 가능하면 공개 DEM/DSM 또는 사용자가 승인한 샘플 데이터만 사용한다. |
| 링크품질 직접 주장 금지 | RSSI/SINR/packet loss 없이 통신품질 보장 표현을 쓰지 않는다. |
| 가중치 고정값 경계 | 0.80/0.20, 0.50/0.35/0.15는 초기 휴리스틱으로만 기록한다. |
| 민감도 분석 필수 | 가중치를 바꿨을 때 색상지도와 경로 후보가 얼마나 변하는지 비교한다. |
| 중복 반영 점검 | DSM이 LOS/Fresnel 계산에 이미 쓰이면 DSM 장애물 점수의 중복 위험을 점검한다. |
| 논문 표현 제한 | 성능 입증이 아니라 오프라인 방법론 검증으로 표현한다. |

## 4. 단계별 보완계획

## Stage 0. 기준선 고정

### 목적

현재 점수식을 baseline, 즉 기준선으로 고정한다.

### 사용 데이터

- 점수식 정의
- 가중치 설정값
- synthetic 또는 샘플 입력 파라미터

### 작업

1. 점수식 이름을 명확히 한다.
   - `overall_launch_score`
   - `shielding_stability_score`
   - `distance_score`
   - `los_score`
   - `fresnel_margin_score`
   - `dsm_obstacle_score`
2. 모든 점수 범위를 0~100으로 통일한다.
3. 100점은 좋음, 0점은 나쁨으로 방향성을 통일한다.
4. 가중치 합이 1.0인지 검증한다.
5. `config.yaml` 또는 `config.py`에서 가중치를 바꿀 수 있도록 설계한다.
6. 실제 링크 측정값 없이도 계산 가능한 값만 입력으로 받도록 schema를 제한한다.

### 산출물

- baseline score configuration
- 점수 범위 정의표
- 가중치 합 검증 테스트
- 오프라인 입력 데이터 schema

---

## Stage 1. 항목별 정규화 검증

### 목적

LOS, Fresnel, DSM, 거리점수가 서로 비교 가능한 점수인지 확인한다.

### 사용 데이터

- synthetic 지형 단면
- synthetic DSM 장애물
- 목표점·후보점 좌표
- 주파수 대역
- AGL/MSL 계산값
- 운용반경

### 작업

1. LOS 점수
   - 완전 가시: 100
   - 일부 차단 또는 경계 차단: 50 이하 후보
   - 완전 차단: 0
2. Fresnel 여유 점수
   - 제1 Fresnel Zone 침범률 0%: 100
   - 침범률 20% 이하: 제한 감점
   - 침범률 40% 이상: 큰 감점
   - 침범률 100% 또는 완전 차단: 0
3. DSM 장애물 점수
   - DSM 표면장애물 없음: 100
   - 장애물 높이가 LOS/Fresnel 경계에 근접: 중간 감점
   - 장애물이 직접 차폐: 0 또는 강한 감점
4. 거리점수
   - 목표까지 3D 거리 0: 100
   - 운용반경 한계: 0
   - 운용반경 초과: 제외 처리
5. 모든 점수항에서 실제 RSSI, SINR, packet loss를 요구하지 않는지 확인한다.

### 산출물

- 항목별 점수 변환표
- 경계값 테스트 케이스
- 점수 방향성 테스트
- 실제 링크데이터 미사용 확인표

---

## Stage 2. Synthetic 지형 단위검증

### 목적

실제 드론 운용 없이 가상 지형에서 점수식이 직관과 맞는지 확인한다.

### 사용 데이터

- synthetic DEM
- synthetic DSM
- synthetic 목표점·후보점
- synthetic 운용반경·AGL·주파수 조건

### synthetic 지형 시나리오

| 시나리오 | 설명 | 기대 결과 |
|---|---|---|
| S1 평탄지 | 장애물 없는 평지 | LOS/Fresnel 점수 높음, 거리 중심으로 색상 변화 |
| S2 단일 능선 | 목표와 후보점 사이에 능선 배치 | 능선 뒤쪽 점수 하락 |
| S3 협곡/계곡 | 양쪽 고지와 저지 경로 구성 | 계곡 내부 차폐위험 증가 또는 경로 우회 발생 |
| S4 건물 DSM | DEM은 평탄하지만 DSM 건물 존재 | DSM 반영 시 점수 하락 |
| S5 수목 DSM | 낮은 장애물 다수 | Fresnel/DSM 여유도에 따른 완만한 감점 |
| S6 운용반경 경계 | 같은 차폐조건이나 거리만 증가 | 반경 경계 근처 색상 하락, 초과 지역 제외 |
| S7 극단 고도 | AGL을 낮게/높게 변경 | AGL 상승 시 차폐위험 감소 경향 |

### 산출물

- 시나리오별 점수 분포
- 색상지도 변화표
- 경계조건 통과/실패 기록
- 오프라인 검증용 그림·표 후보

## Stage 3. DSM 중복 반영 점검

### 목적

DSM 표면고도를 LOS와 Fresnel 계산에 이미 사용했을 때, DSM 장애물 점수가 같은 장애물을 두 번 벌점화하는지 확인한다.

### 사용 데이터

- 동일한 synthetic DEM
- 동일한 synthetic DSM
- 동일한 목표점·후보점 세트
- 동일한 AGL·주파수 조건

### 비교 모드

1. DEM-only 모드
   - LOS/Fresnel은 DEM만 사용
   - DSM 장애물 점수는 별도 보정항으로 사용
2. DSM-integrated 모드
   - LOS/Fresnel 계산에 DSM 사용
   - DSM 장애물 점수는 0.15에서 0.05 이하로 낮추거나 제거
3. Hybrid 모드
   - LOS는 DEM 기준
   - Fresnel 침범은 DEM/DSM 중 선택 가능
   - DSM 점수는 근거리 표면장애물 밀도 보정으로만 사용

### 권장 MVP 선택

초기 MVP에서는 Hybrid 모드를 권장한다.

```text
LOS: DEM 기반 지형 차폐
Fresnel: DEM/DSM 선택 가능
DSM 점수: 별도 장애물 밀도 또는 표면장애물 보조 지표
```

### 산출물

- DEM-only / DSM-integrated / Hybrid 비교표
- DSM 점수 중복 여부 판정표
- 최종 선택안과 이유
- 논문 한계 문장 후보

## Stage 4. 가중치 민감도 분석

### 목적

0.80/0.20, 0.50/0.35/0.15 가중치가 바뀌어도 결과가 지나치게 불안정하지 않은지 확인한다.

### 사용 데이터

- Stage 2 synthetic 시나리오 결과
- Stage 6 공개/샘플 DEM·DSM 오프라인 결과
- 동일한 격자·동일한 입력 파라미터

### 실험 범위

발진 가능구역 종합점수 가중치:

| 케이스 | 차폐안정성 | 거리 |
|---|---:|---:|
| O1 | 0.90 | 0.10 |
| O2 | 0.80 | 0.20 |
| O3 | 0.70 | 0.30 |
| O4 | 0.60 | 0.40 |

차폐안정성 내부 가중치:

| 케이스 | LOS | Fresnel | DSM |
|---|---:|---:|---:|
| S1 | 0.60 | 0.30 | 0.10 |
| S2 | 0.50 | 0.35 | 0.15 |
| S3 | 0.45 | 0.45 | 0.10 |
| S4 | 0.40 | 0.40 | 0.20 |
| S5 | 0.50 | 0.50 | 0.00 |

### 비교지표

1. 녹색/노란색/주황색/적색/제외구역 면적 비율
2. 색상 등급이 바뀐 셀 비율
3. 상위 가능구역 중심의 공간 이동거리
4. 경로 후보 3개의 총 비행거리 변화
5. 경로 후보 3개의 평균 차폐위험 proxy 변화
6. 경로 후보가 서로 충분히 다른지 여부

### 산출물

- 가중치별 색상 분포표
- 가중치별 경로 비교표
- 민감도 히트맵 후보
- 논문용 가중치 안정성 설명

## Stage 5. Ablation 분석

### 목적

각 점수항이 실제로 필요한지 확인한다.

### 사용 데이터

- synthetic DEM/DSM 시나리오
- 공개/샘플 DEM/DSM 오프라인 시나리오
- 동일한 목표점·후보점·AGL·주파수 조건

### 실험 케이스

| 케이스 | 설명 |
|---|---|
| A0 | 전체 점수식 사용 |
| A1 | 거리점수 제거 |
| A2 | LOS 점수 제거 |
| A3 | Fresnel 점수 제거 |
| A4 | DSM 점수 제거 |
| A5 | LOS만 사용 |
| A6 | LOS + Fresnel만 사용 |
| A7 | 거리만 사용 |

### 기대 해석

- 거리만 사용하면 차폐지형을 충분히 반영하지 못할 수 있다.
- LOS만 사용하면 Fresnel 여유 부족 지역을 과대평가할 수 있다.
- DSM 제거 시 건물·수목 등 표면장애물 영향이 약해질 수 있다.
- 전체 점수식이 가장 균형적이면 현재 구조 유지 근거가 된다.

### 산출물

- 점수항 제거별 색상 분포 변화
- 점수항 제거별 경로 변화
- 각 점수항의 필요성 설명

---

## Stage 6. 공개/샘플 DEM·DSM 오프라인 검증

### 목적

실제 드론 운용 없이 공개 또는 사용자 승인 샘플 지형자료로 재현성을 확인한다.

### 허용 데이터

- 공개 DEM
- 공개 DSM
- 사용자가 명시적으로 제공한 샘플 DEM/DSM
- 저장소에는 대형 원천 데이터를 커밋하지 않고 경로와 메타데이터만 기록

### 금지 데이터

- 실제 비행 로그
- 실제 조종기 로그
- 실제 통신 장비 측정 RSSI/SINR/packet loss
- 실제 운용 성공/실패 결과
- 비공개·비인가 지형자료

### 작업

1. 샘플 지역 1~3개 선정
2. 동일 목표점과 동일 운용반경 설정
3. AGL을 2~3개 수준으로 변경
4. 주파수 대역을 2~3개 수준으로 변경
5. 발진 가능구역 색상지도 생성
6. 경로 후보 3개 생성
7. synthetic 결과와 경향 비교
8. 실제 링크성능이 아니라 오프라인 지형 차폐위험 결과임을 기록

### 산출물

- 샘플 지역별 결과표
- AGL 변화 민감도
- 주파수 변화 민감도
- 공개/샘플 지형자료 사용 한계
- 실제 링크상태 검증 미수행 기록

## Stage 7. 전파모델 대조 검증

### 목적

현재 점수식이 실제 전파 손실 모델은 아니므로, Longley-Rice/ITM, ITU terrain diffraction, free-space loss 등과 개념적으로 일치하는지 오프라인 대조한다.

### 사용 데이터

- 거리
- 주파수
- LOS 차단 여부
- Fresnel 침범률
- DSM 장애물 존재 여부
- 지형 단면

### 방법

1. 현재 점수식 결과를 dB 손실로 단정 변환하지 않는다.
2. 후보 링크 몇 개를 선택한다.
3. 각 링크에 대해 다음 지표를 비교한다.
   - 거리
   - LOS 차단 여부
   - Fresnel 침범률
   - DSM 장애물 존재 여부
   - 자유공간손실 상대값
   - 지형회절 위험 proxy
4. 전파모델과 방향성이 같은지 확인한다.
5. 실제 수신전력, RSSI, SINR, packet loss와 비교한 것처럼 쓰지 않는다.

### 논문 표현

```text
본 점수는 전파수신전력 예측값이 아니라 지형 및 표면장애물 기반 차폐위험 proxy이다. 실제 수신전력 또는 패킷 손실률과의 정량 검증은 수행하지 않았으며, 향후 연구로 둔다.
```

## Stage 8. 전문가/사용자 검토

### 목적

실제 비행 없이 군사지형·전파·GIS 관점에서 점수식의 상식적 타당성을 검토한다.

### 검토 질문

1. 차폐안정성을 거리보다 우선하는 0.80/0.20 구조가 목적에 맞는가?
2. LOS 0.50, Fresnel 0.35, DSM 0.15의 상대적 중요도가 설명 가능한가?
3. DSM을 독립 점수로 둘 경우 중복 반영 위험이 수용 가능한가?
4. 색상 등급 임계값이 과도하게 낙관적 또는 보수적인가?
5. 결과물이 의사결정 보조 수준으로 표현되고 있는가?
6. 실제 드론운용 없이 가능한 검증 범위를 넘어서 주장하지 않았는가?

## Stage 9. 논문 반영 기준

논문에는 다음 수준으로 반영한다.

| 검증 수준 | 논문 표현 |
|---|---|
| synthetic 검증 | 알고리즘 동작 및 경계조건 검증 |
| 공개 DEM/DSM 검증 | 지형자료 기반 오프라인 재현성 확인 |
| 전파모델 대조 | 점수 방향성 비교 |
| 실제 링크 측정 없음 | 한계 및 향후 연구 |
| 실제 드론 운용 없음 | 연구·교육·시뮬레이션용 방법론 |

## 5. 최종 권고

MVP 단계에서는 현재 가중치를 유지하되 다음을 반드시 적용한다.

1. 가중치를 설정파일로 분리한다.
2. 점수 범위와 방향성을 0~100으로 통일한다.
3. DSM 중복 반영 여부를 테스트한다.
4. 민감도 분석과 ablation 분석을 Task 012~014에서 수행한다.
5. 논문에는 현재 가중치를 검증된 최적값이 아니라 초기 휴리스틱으로 표현한다.
6. 실제 드론 운용 없이 synthetic + 공개/샘플 DEM/DSM + 전파모델 대조 검증만 수행한다.
7. RSSI, SINR, packet loss, 실제 통신 성공률은 산출 대상에서 제외한다.
8. 모든 실험 로그에 `actual_drone_operation = false`, `actual_link_measurement = false`를 기록한다.

## 6. Cloud Execution Agent 구현 지시 기준

Cloud Execution Agent가 점수식 관련 Task를 수행할 때는 다음 조건을 지킨다.

1. 실제 드론운용 데이터를 요구하지 않는다.
2. 실제 링크 측정값을 입력 schema에 필수값으로 넣지 않는다.
3. synthetic DEM/DSM으로 재현 가능한 테스트를 우선 작성한다.
4. 공개/샘플 DEM·DSM은 경로와 메타데이터만 기록하고 원천 대형 데이터는 커밋하지 않는다.
5. 출력값 이름에는 `proxy`, `risk`, `score`, `offline` 등 표현을 사용하고 `guaranteed`, `verified link quality`, `communication success` 표현은 사용하지 않는다.
6. 논문 기록에는 실제 드론운용 없이 산출된 오프라인 분석 결과임을 명시한다.
